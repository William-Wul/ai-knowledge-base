# 建设历程

一个非技术 PM(William)花了两个多月,在 Claude 的协助下搭起的公司内部 AI 学习知识库。
这份文档按时间顺序记录每个关键节点:**做了什么、当时卡在哪、最后怎么过去**。
写在这里,不是为了给别人看,是以后自己回头翻能想起"其实做了挺多事"。

---

## 2026 年 4 月初 · v1.0 起步

决定做这件事的初衷:公司 300 来号人里绝大多数是非技术背景,AI 一天一个新词,大家想用但不知道从哪儿下手。
现成的 AI 课程要么太浅(只教你怎么用 ChatGPT),要么太深(一上来就讲 Transformer),于是决定自己搭一个面向非技术员工的学习路径。

第一版选型:

- **VitePress** 写文档、自动生成静态网站(只要会写 Markdown 就能改内容)
- **GitHub Pages** 免费托管
- **Netlify Identity** 做访问控制(只给公司同事看)

跑通第一版后,整站用密码门保护,有了最基本的形态。

## 4 月 21 日 · v1.3 · 一个差点毁站的 P0 Bug

登录完进任何文章,侧边栏正常、"上一篇/下一篇"正常,就是**正文区域完全空白**。
整站等于废了。

花了半天定位,根因是:
**PasswordGate 组件做成了 VitePress Layout 的父级包裹器**。结果 VitePress 的路由数据加载时机错位,文章的主内容 chunk 根本没被加载。

解法是把 PasswordGate 改成和 Layout **平级**的覆盖层组件(`position: fixed` 遮罩),让 Layout 从初始渲染就挂载。
文章内容就回来了。

这次之后立了一个硬规矩:**任何代码改动上线前必须跑 `npm run docs:build` 验证**,而不是只在本地 `npm run docs:dev` 下看着正常就发。

## 4 月 23 日 · v1.6 · AI 热点专区 + 每天自动更新(今天)

整套自动化上线。每天北京时间 **21:00**,GitHub Actions 会自动跑完整条流水线:

1. 克隆开源热榜聚合工具 **TrendRadar**(固定到验证过的版本)
2. 抓取 11 个国内热榜(微博/知乎/B 站/头条/百度/澎湃/抖音/贴吧/华尔街见闻/财联社/凤凰网)+ Hacker News RSS
3. 过滤出 AI 相关条目(关键词表分三层:核心层 / 玩家层 / 应用层)
4. 调用 **OpenRouter → DeepSeek Chat** 做两件事:把跨平台的同一事件合并去重、为每条事件综合打分(重要性 + 时效性 + 讨论度 + 影响面)并写 200-300 字概述
5. 生成当日 MD,commit 回仓库,触发站点重新部署

单次成本约 $0.002(约 1 分 4 厘人民币),一年总计 ~$0.73。
OpenRouter API key 存在 GitHub Secrets 里,不会出现在代码。

前端设计上做了几件事:

- **每条热点默认折叠**,只显示标题 + 🔥 数量(1-5 档),点击展开才看概述
- **不显示数字分数**(读者看不懂 `73/100` 是什么意思),只靠 🔥 数量传达热度直觉
- **排序完全按 LLM 综合判断**,不是机械的"平台数 × 排名"公式

设计过程中调整过几次:

- 最初 frequency_words.txt 用了 `/\bAI\b/` 正则,结果"用AI办了场车展"这种中文紧贴标题一条没抓到 → 改成普通关键词 `AI`,命中数从 3 翻到 6
- RSSHub 公共实例抓 GitHub Trending 和 Product Hunt 都被限流 403 → 降级只保留 Hacker News,记为待办
- 入口页最初把所有技术栈细节(OpenRouter / GitHub Actions / 11 个平台清单 / 关键词维护说明)都暴露给读者 → 精简为"读者只看到 🔥 和内容",所有内部字段删光
- LLM 最初按自己理解的"重要性"排序但显示的热度分是机械算的,导致排名 1(20 分)在排名 2(23 分)前面,视觉翻车 → 干脆把打分和排序一起交给 LLM,前端只显 🔥 不显数字

验证时遇到的插曲:本地预览端口 4567 反复 404 → 原因是 VitePress 的 `preview` 命令启动时只读了那一刻的路由清单,新加的 MD 文件要重启才认。后来知道这是本地临时问题,GA 自动化后不存在。

## 一路上踩过的坑(硬约束,以后不踩第二次)

- **只跑 dev 不跑 build,线上部署失败过两次** → 铁规:提交前必过 `npm run docs:build`
- **base 路径要跟着域名走** → 从 `/ai-knowledge-base/` 迁到自定义域名 `ailinkstart.com` 时,配置改成 `/`,忘改就 404
- **Cloudflare Proxy 橙色云朵开启时,GitHub Pages "Enforce HTTPS" 会灰色** → 不是坏了,是正常现象,保持 Proxy 开启即可
- **publish.sh 用 `git add .`** → 任何 untracked 工作文件都会进仓库,`.gitignore` 必须先写好
- **Netlify 账号因 credit limit 被暂停,Decap CMS 后台登录废了** → 暂时搁置,近期重点在内容;网站本身不受影响
- **公开仓库安全警觉** → `CLAUDE.md` 和 `kb-articles/` 曾经误入仓库(commit d35ef10),v1.5.1 补了 .gitignore 收紧

## 一个非技术 PM 的感想

两个月前还不知道什么是 VitePress、什么是 GitHub Actions、什么是 secrets。
现在可以自己描述需求、自己决策技术选型、自己验证效果。
没写一行代码,但能把系统搭起来、能让它每天自动跑、能出问题时知道去哪看日志。

AI 时代的"协作力",大概就是这样一种:**你不用会写,但你要会想、会问、会判断**。

这份知识库想让公司的同事们也具备这个能力。
自己先走一遍这条路,才更清楚下面怎么教。
