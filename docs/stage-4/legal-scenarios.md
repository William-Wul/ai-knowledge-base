# 法务场景

> 本文预计阅读时间：12 分钟 &nbsp;·&nbsp; <span class="stage-badge stage-3">阶段三 · 场景深耕</span>

---

法务工作对准确性要求极高，AI 在这里的定位是**辅助工具而非法律顾问**。它能帮你快速梳理思路、整理清单、解释条款，但涉及真实法律判断和风险决策，必须由专业人员确认。

> **重要提醒**：不要把未签署的合同原文、商业秘密条款、正在进行的诉讼详情输入 AI。可以用改写后的通用描述代替具体信息。

---

## 场景一：整理合同审查检查清单

每次审合同都从头想条款，效率低。让 AI 帮你生成一个通用的合同审查清单，作为固定工具。

**模板：**
```
请帮我生成一份【合同类型，例如：采购合同 / 服务合同 / 劳动合同】的审查检查清单，要求：
- 包含常见的风险点（按重要程度排序）
- 每个风险点说明"要看什么"和"常见的问题是什么"
- 格式：分条列出，便于逐项勾选
- 重点关注：付款条款、违约责任、争议解决、知识产权归属
```

---

## 场景二：解释合同中看不懂的条款

合同里经常有晦涩的法律术语，AI 能用大白话解释。

**模板：**
```
请用通俗易懂的语言解释以下合同条款的含义，以及它对我方（作为【甲方/乙方】）意味着什么风险或义务：

条款内容（已做通用化处理）：
"【把条款文字粘贴进来，建议先删掉公司名称、金额等敏感信息】"

请说明：
1. 这条款的核心意思是什么
2. 对我方有什么实际约束
3. 有没有需要特别注意或谈判的点
```

## 场景三：起草合同审查意见摘要

审完合同，需要给业务部门出一份简单易懂的意见摘要。

**模板：**
```
我已经审阅了一份合同，发现以下几个需要关注的点（已脱敏）：

1. 【问题一描述，例如：付款周期约定为 90 天，超出我方通常接受的 60 天标准】
2. 【问题二描述，例如：违约金条款仅约定了对方违约的赔偿，未约定我方违约的上限】
3. 【问题三描述，例如：争议解决选择了仲裁，地点在对方所在城市】

请帮我把这几个点整理成一份合同审查意见摘要，格式要求：
- 语言简洁，让没有法律背景的业务同事能看懂
- 每个问题说明风险级别（高/中/低）
- 给出建议的处理方式（保留/修改/谈判）
```

## 场景四：查询法规政策基本信息

遇到不熟悉的法律法规，先用 AI 建立基础认知，再查官方文件确认。

**模板：**
```
请帮我介绍【法规名称，例如：个人信息保护法 / 劳动合同法第 X 条】的基本内容：
1. 这个法规/条款的核心要求是什么
2. 对企业的主要影响和义务是什么
3. 常见的违规情形有哪些
4. 我需要去哪里查阅官方原文

注意：我知道 AI 的回答仅供参考，最终以官方文件为准。
```

---

## 场景五：整理常见法律问题 FAQ

业务部门经常来问同类问题，提前用 AI 整理成 FAQ 文档，减少重复解答。

**模板：**
```
请帮我整理一份关于【主题，例如：员工劳动关系 / 数据合规 / 采购合同】的常见法律问题 FAQ，要求：
- 列出 8-10 个业务部门最常问的问题
- 每个问题给出简明的解答要点（200 字以内）
- 在解答末尾注明"如涉及具体情况，请联系法务确认"
- 使用非技术化语言，让没有法律背景的同事能看懂
```

---

## 场景六：起草法务相关的内部通知

**模板：**
```
请帮我写一则内部法务通知，内容如下：
- 通知主题：【例如：关于规范对外合同签署流程的通知】
- 核心内容：【例如：所有对外合同金额超过 10 万元须经法务审核，审核时间为 3 个工作日】
- 生效日期：【例如：即日起】
- 语气：正式严谨，但不要全是官方套话
- 格式：标准通知格式，带标题和落款
```

---

## 法务场景的 AI 使用边界

| ✅ AI 能帮你做的 | ❌ 必须由人来做的 |
|---------------|----------------|
| 生成合同审查检查清单 | 对具体合同的法律风险判断 |
| 解释条款的通用含义 | 对特定合同条款的法律效力认定 |
| 整理法规的基本信息 | 出具正式的法律意见 |
| 帮你梳理审查意见的表达 | 代表公司与对方进行法律谈判 |
| 整理内部 FAQ 文档 | 诉讼案件的策略判断 |

---

> **一句话原则**：AI 帮你整理思路、提高效率；法律判断，人来负责。
