## 一、一句话摘要

Kimi K2.6 是**月之暗面**最新发布的开源大模型，官方跑分直接对标 GPT-5.4、Claude Opus 4.6、Gemini 3.1 Pro，在 **Agent、代码、视觉** 三大维度上与闭源顶流正面掰手腕。作者的观点转变：推荐国产模型的理由，**从"便宜"变成了"又便宜又能打"**。

## 二、核心要点（Key Takeaways）

1. **Agent 能力全面领先**：HLE（人类最后的考试）带工具成绩超过 GPT-5.4 和 Claude Opus 4.6；浏览器自动操作、深度搜索、OS 级任务稳居第一梯队。
2. **代码能力反超闭源**：SWE-Bench Pro 超 GPT-5.4 约 2%，比 Claude Opus 4.6 高近 10%。
3. **视觉理解大幅进步**：数学视觉推理比 Claude Opus 4.6 高近 10 个百分点，与 GPT-5.4 差距仅 3 点。
4. **长程任务不崩盘**：Agent 可连续自主运行 5 天无人类介入；代码任务可连续干 13 小时、发起 4000+ 工具调用。
5. **Agent Swarm 规模拉 3 倍**：从 K2.5 的 100 个子 Agent / 1500 步提升到 K2.6 的 **300 个子 Agent / 4000 步**。
6. **开源 + 免费**：所有用户可直接用，已上线 Kimi API 和 Kimi Code。
7. **明确短板**：纯数学推理仍有差距；与 Anthropic 刚发布的 Opus 4.7 的竞争远未结束。

## 三、分章节要点梳理

### 01｜成绩单：赢在哪，差在哪

作者把官方几十项跑分按三类划重点：

| 维度 | K2.6 表现 | 对比 |
|---|---|---|
| **Agent（通用智能体）** | 全面领先 | HLE 带工具成绩超过 GPT-5.4 和 Claude Opus 4.6 |
| **代码** | 反超闭源 | SWE-Bench Pro 超 GPT-5.4 约 2%，比 Claude Opus 4.6 高近 10%；终端编程与 GPT-5.4 打平 |
| **视觉理解** | 大幅拉开差距 | 数学视觉推理比 Claude Opus 4.6 高近 10 点，与 GPT-5.4 差 3 点 |

作者态度诚实：**"不是全面碾压"**，但开源模型在代码和 Agent 正面刚闭源顶流，"这事本身就挺了不起"。

### 02｜代码能力：连续干 13 小时不崩

作者强调 K2.6 的"代码能力"与常规理解不同——**不是补函数、修 Bug，而是自主规划 + 自主迭代 + 长程执行**。

**案例 A：Zig 语言 + Qwen3.5-0.8B 推理优化**
- 没人教过它 Zig，完全自学
- 连续跑 12 小时、4000+ 工具调用、迭代 14 个版本
- 推理速度 **15 → 193 tokens/秒**，比 LM Studio 快 20%

**案例 B：8 年老项目 exchange-core（开源金融撮合引擎）**
- 接手一个被优化到极限的项目
- 连续 13 小时、1000+ 工具调用、改 4000+ 行代码
- 中位吞吐量 **+185%**

作者一句值得摘录的原话：
> "以前觉得 AI 写代码是'帮你省时间'，现在看来，它更接近'替你干那些你不想干、或者干不了的重活累活'。"

### 03｜养虾（OpenClaw）用户：该换大脑了

- 背景：OpenClaw 的 PinchBench 测试中，K2.5 就已是官方推荐最佳适配之一（仅次于 Gemini 3 Flash，与 MiniMax 并列）
- K2.6 的核心进步 **不是"更聪明"，而是"更稳"**
- 官方让 K2.6 驱动的 Agent **自主运行 5 天无人介入**，全程处理监控告警、故障响应、系统运维
- 月之暗面内部有一个专测 OpenClaw 场景的 **Claw Bench**，覆盖：编程 / 飞书生态对接 / 信息搜索 / 定时任务 / 记忆调用 五个维度。K2.6 综合提升约 **10%**
- 适用人群：OpenClaw、Hermes 用户可考虑切换

### 04｜Agent 集群：300 个分身同时干活

- **规模升级**：K2.5 → Agent Swarm，100 个子 Agent / 1500 步
- **K2.6 → 300 个子 Agent / 4000 步**，扩大 3 倍
- 典型案例：
  - 上传一份简历 → 拆出 100 个子任务 → 每个 Agent 匹配一个岗位 → 一次性生成 **100 份定制简历**
  - 分析 100 家半导体公司 → 5 套量化策略 + 麦肯锡风格 PPT + 建模表格
- **Claw Groups（研究预览版）**：多 Agent 组队，可跨设备、跨模型、跨技能，K2.6 当"包工头"负责分配 / 监控 / 异常处理
- 作者形容：**"AI 时代的'项目经理'"**

### 05｜其他升级快览

|---|---|
| **代码驱动设计**（一句话建网站） | 给一句描述出带动效带交互的完整网站；可做简单全栈应用（数据库+登录）；设计稿/手绘草图也能还原 |
| **Office 能力** | Excel 写复杂公式做建模；Word 插批注做合同审阅；PPT 丢论文出 20 页答辩幻灯片；格式间无损互转 |
